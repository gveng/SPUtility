SPUtility User Guide

SPUtility is a desktop viewer and analysis tool for Touchstone S-parameter files. This guide explains both how to use each section and how calculations and measurements are performed.

Main Workflow

  1. Use File > Open File to load one or more Touchstone files.
  2. Use Tables to inspect raw complex values and magnitude values.
  3. Use Charts > Open plot window for S-parameter overlays.
  4. Use TDR > Open TDR window for time-domain impedance profiles.
  5. Use Circuit > Open circuit composer for network assembly, export, and channel simulation.
  6. Use File > Save Project to store full workspace state.

File Menu and Recent History

The File menu contains two independent recent histories:

  • Recent Projects: recently opened or saved project JSON files.
  • Recent S-Parameters: recently opened Touchstone files (.sNp, .ts).

Missing files are removed automatically from history when selected.

S-Parameter Plot Windows

What is calculated

  • The plotted Y values come from the precomputed magnitude table in dB.
  • Each selected trace is drawn versus frequency.
  • Optional log axes filter out non-valid points for the chosen scale.

How measurements are detected

  • Plot windows do not estimate extra derived SI metrics by default.
  • They report the selected traces exactly as present in loaded data tables.
Magnitude in dB: 20 * log10(|Sij|)

TDR Window

What is calculated

  1. Select a reflection trace Sii(f) (for example S11).
  2. Sort frequencies and resample only if spacing is non-uniform.
  3. Apply selected high-frequency taper window.
  4. Compute impulse response by inverse real FFT.
  5. Integrate impulse to obtain step reflection rho_step(t).
  6. Convert reflection to impedance Z(t).
  7. Apply optional display processing: rise-time smoothing, front shift, total-time clipping.
rho_step(t) = cumsum(irfft(window * Sii(f))) Z(t) = Z0 * (1 + rho_step(t)) / (1 - rho_step(t))
TDR conversion chain used in SPUtility.

How measurements are detected

  • The tool computes the full impedance-vs-time waveform.
  • Displayed values are direct waveform samples after selected processing settings.

Circuit Editor and Equivalent Export

What is calculated

  • The editor builds a network graph from blocks, ports, and connections.
  • For each sweep frequency, an equivalent network solve is executed.
  • The exported S-matrix is written to Touchstone in the selected format and unit.
Per-frequency solve: Y * V = I Typical conversion form: S = (I - Z0*Y) * inv(I + Z0*Y)
From circuit graph to exported Touchstone model.

How measurements are detected

  • Validation checks topology consistency and sweep constraints before export.
  • External ports are detected from dedicated port blocks and rebuilt automatically.

Channel Simulation and Eye Diagram

What is calculated

  1. Generate transmitted waveform from driver settings (levels, bitrate, encoding, PRBS, rise/fall).
  2. Run channel response through the assembled network.
  3. Fold waveform into UI-length segments and align decision phase.
  4. Optionally apply statistical perturbation: Gaussian voltage noise and random timing jitter.
  5. Render eye with selected mode (Heatmap / Lines / Heatmap + Lines) and quality preset (Fast / Balanced / HighRes).
UI = 1 / Bitrate Jitter (samples) = jitter_rms_ps * 1e-12 / dt

How measurements are detected

  • One Level / Zero Level: medians of upper/lower amplitude clusters around the decision center.
  • Eye Amplitude: One Level - Zero Level.
  • Eye Height: vertical opening at decision center using percentile envelopes.
  • Eye Width: contiguous open interval around center, reported in ps.
  • Eye Crossing: crossing level measured around +/-0.5 UI and normalized to Eye Amplitude (%).
  • Crossings are aligned around -0.5 UI and +0.5 UI on the 2-UI display span.
EyeAmplitude = OneLevel - ZeroLevel EyeHeight = p5(upper) - p95(lower) at decision center EyeWidth_UI = open_samples / samples_per_UI EyeWidth_ps = EyeWidth_UI * UI * 1e12 EyeCrossing_pct = 100 * (CrossingLevel - ZeroLevel) / EyeAmplitude
Eye measurement geometry used by the summary panel.

Project Files

A saved project stores:

  • Loaded file paths.
  • Plot, TDR, and circuit windows with their settings and sizes.
  • Selected traces, legend labels, and per-window configuration.

View Menu

  • Tile windows: arrange sub-windows in a tiled layout.
  • Cascade windows: stack sub-windows in cascade order.
  • Minimize all: minimize every open sub-window.
  • Restore all: restore all minimized sub-windows.
  • Resize all graph with last graph window: copy layout from latest graph window.
  • Close all: close every open sub-window.

Notes

  • If a saved project references missing files, loading proceeds with warnings.
  • You are prompted before closing when the project has unsaved changes.
