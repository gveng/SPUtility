SPUtility User Guide

SPUtility is a desktop viewer, network composer and channel-simulation tool for Touchstone S-parameter files. It targets signal-integrity engineers who need a quick, scriptable, single-window workflow to inspect S-parameter blocks, build interconnect models, run time-domain reflectometry, convolve transmitted bit streams through linear channels, and read out eye-diagram metrics.

This document describes the GUI and the underlying numerical methods, so that every measurement and every plot can be reproduced and audited. It corresponds to version 2.2.

1. Main Workflow & Quickstart ↑ top

  1. Use File > Open File to load one or more Touchstone files (.s2p, .s4p, .s8p, …).
  2. Use Tables to inspect raw complex values and magnitude values.
  3. Use Charts > Open plot window for S-parameter overlays.
  4. Use TDR > Open TDR window for time-domain impedance profiles.
  5. Use Circuit > Open circuit composer for network assembly, export, transient and channel simulation.
  6. Use the project tree on the left to reopen, minimize, duplicate, or delete existing items.
  7. Use File > Save Project to store full workspace state.
  8. Use File > Export Project when you need a portable ZIP bundle with copied project data and referenced Touchstone sources.

2. File Menu, Recent History & Project Export ↑ top

The File menu contains two independent recent histories:

  • Recent Projects: recently opened or saved project JSON files.
  • Recent S-Parameters: recently opened Touchstone files (.sNp, .ts).

Missing files are removed automatically from history when selected.

Export Project creates a sibling folder named <ProjectName>_Export, copies the saved project, duplicates generated project data into its local data folder, gathers referenced Touchstone sources, and finally creates a ZIP archive. The exported project can then be moved to another machine without manually rebuilding the folder structure.

3. Project Tree & Window Management ↑ top

The left project tree is the main control point for already loaded files, open analysis windows, and generated outputs. File extensions are hidden in the tree to keep names readable, while item icons identify Touchstone files, S-parameter plots, circuits, and eye diagrams.

Analysis windows are no longer hosted inside an MDI workspace. Each output category (Circuits, S-Parameter Plots, TDR, Eye Diagrams, Transient Results, Tables) lives in its own top-level window with tabs, so it is independently movable, resizable and visible on the Windows taskbar with the icon of the active tab. A tab can be popped out into a standalone window via right-click → Detach to separate window (or by double-clicking the tab); each detached window appears as its own taskbar entry as well.

4. View Menu ↑ top

  • Minimize all / Restore all: bulk minimize/restore every category window and detached window.
  • Resize all graph with last graph window: copy layout from latest graph window.
  • Close all: close every open category window and detached window.

5. Tables Window ↑ top

The Tables window exposes the parsed Touchstone data without any further processing. Two views are available: a complex view (real / imaginary or magnitude / phase as stored) and a magnitude in dB view (precomputed once at parse time as 20·log10(|Sij(f)|)). The reference impedance shown in the header is taken from the Touchstone ! Z0 = directive when present (defaulting to 50 Ω).

6. S-Parameter Plot Window ↑ top

6.1 Theory: S-parameters & reference impedance

For an N-port linear time-invariant network at angular frequency ω = 2πf, incident and reflected normalized waves at port k are

a_k = (V_k + Z0_k * I_k) / (2 * sqrt(Re{Z0_k})) b_k = (V_k - Z0_k * I_k) / (2 * sqrt(Re{Z0_k}))

The scattering matrix relates them as b = S·a. Each entry of S can be interpreted as:

  • Sii: voltage reflection at port i when all other ports are matched.
  • Sji: voltage transmission from port i to port j when all other ports are matched.

Plot windows show the magnitude of the selected entries in dB:

|Sij|_dB = 20 * log10(|Sij(f)|)

6.2 Axes, scaling and source management

  • X axis: linear or logarithmic frequency. Logarithmic mode filters non-positive frequencies.
  • Y axis: linear (dB) or logarithmic dB; the latter is useful for return-loss tails.
  • Each loaded Touchstone file appears in the per-plot file list. Right-click a file:
    • Unload file — removes the file from the whole application state and updates every dependent window.
    • Remove from this plot — hides the file in the current plot only; the file is still available to other windows and the saved project.
  • Plot legend labels can be edited in place to produce publication-ready figures.

7. TDR Window ↑ top

7.1 Theory: Time-Domain Reflectometry

TDR converts a frequency-domain reflection coefficient Sii(f) into the time-domain voltage reflection observed at the same port when an ideal step is launched into the line. For a linear network the relation is purely a Fourier-transform pair plus a step-integration:

h(t) = F^-1 { Sii(f) } (impulse reflection) rho_step(t) = integral_{-inf..t} h(tau) d tau (step reflection) Z(t) = Z0 * (1 + rho_step(t)) / (1 - rho_step(t))
TDR conversion chain used in SPUtility.

7.2 Windowing & uniform resampling

The inverse FFT requires uniform frequency spacing and assumes the spectrum is band-limited. SPUtility:

  1. Sorts frequency points and detects whether spacing is uniform (relative tolerance < 1‰).
  2. If non-uniform, resamples on a uniform [fmin, fmax] grid using monotonic PCHIP interpolation (Piecewise Cubic Hermite Interpolating Polynomial) on real and imaginary parts independently — this preserves monotonic features without Gibbs-style overshoot.
  3. Applies a high-frequency taper window (default Tukey α=0.25) to the upper part of the spectrum. Without windowing, the truncated spectrum produces strong sinc-shaped ringing in the time-domain step.
  4. Pads with zeros up to the next power of two for FFT efficiency and improved time resolution.
dt = 1 / (2 * fmax) (Nyquist sample period) Resolution length = c0 / (2 * fmax) for vacuum; divide by sqrt(eps_r) for line propagation

7.3 Impulse response & step integration

The impulse reflection is computed by an inverse real FFT (numpy.fft.irfft), exploiting the Hermitian symmetry of physical S-parameters. The step reflection is the running cumulative sum:

rho_step[n] = sum_{k=0..n} h[k]

Causality is enforced implicitly: the spectrum is assumed Hermitian (Sii(-f) = Sii*(f)), so h(t) is real. Negative-time leakage smaller than 1‰ of peak is treated as numerical noise.

7.4 Reflection → impedance conversion

Z(t) = Z0 * (1 + rho_step(t)) / (1 - rho_step(t))

This is the standard mapping from voltage reflection coefficient to impedance, valid for a single-mode lossless probe of impedance Z0. Values of |ρstep| approaching 1 are clamped at ±0.999 to avoid division blow-up.

7.5 Display processing

  • Rise time smoothing: Gaussian convolution with σ chosen so that the 10–90% rise of an ideal step matches the requested value (tr ≈ 2.5631·σ). Mimics a real TDR sampler.
  • Front shift: time origin shift to align the launch edge with t = 0.
  • Total time: clip the visible window to the range relevant to the DUT length.

8. Circuit Editor ↑ top

The circuit editor is a schematic canvas that lets you assemble Touchstone blocks together with lumped components, sources, ports and probes, and then run frequency-domain solves, transient simulations or full channel simulations on the resulting network.

8.1 Element library

Each block contributes its own admittance / scattering matrix to the global solver. Below is the complete reference, with symbol, parameters and electrical model.

Touchstone block

Generic N-port block backed by an external Touchstone file. Carries the file's own reference impedance Z0 and frequency grid; the solver reinterpolates it onto the global sweep.

  • Source file: Touchstone file providing S(f).
  • Display label: free text shown on the canvas.
  • Port count: derived from file extension (.sNp).
  • Rotation / mirror: cosmetic only; does not change electrical behaviour.

Resistor (R)

Two-port lumped resistor. Modelled as an ideal series resistance with no parasitics.

  • Resistance R [Ω]: any positive real value.

Two-port admittance contribution Y = (1/R)·[[1,−1],[−1,1]].

Inductor (L)

Two-port lumped inductor. Modelled as an ideal series inductance.

  • Inductance L [H]: positive real.

Series impedance per frequency: Z(f) = j·2πf·L. Admittance contribution: Y = (1/Z)·[[1,−1],[−1,1]].

Capacitor (C)

Two-port lumped capacitor.

  • Capacitance C [F]: positive real.

Series impedance per frequency: Z(f) = 1 / (j·2πf·C).

Ground (GND)

Local ground anchor. Any node connected to a GND symbol is forced to V = 0 in the solver. Used to terminate shunts (R, C, L to ground).

Net node

Multi-way wire junction. Lets you fan out one signal node to several blocks without crossing wires. Electrically transparent.

Single-ended Port

External port of the equivalent network. The block contributes one row/column to the exported S-matrix and defines the reference impedance seen at that port.

  • Port number: index in the exported Touchstone file.
  • Reference impedance Z0 [Ω]: defaults to 50.

Differential Port

Pair of single-ended ports treated as a differential pair. The exported S-matrix contains both legs; mode conversion (mixed-mode S) can be derived externally.

Port-Ground

Defines the ground reference of an external port (used when the source/load reference is not the global circuit ground, e.g. floating shielded measurements).

Driver (single-ended)

Bit-stream voltage source used by the channel simulator. Only meaningful for channel simulations — it does not contribute a passive admittance to the network solver, but its Source impedance is used in the voltage transfer function (see §9.5).

ParameterMeaning
Vhigh, Vlow [V]Logic levels of the transmitted NRZ stream.
Rise / Fall time [s]10–90 % transition times. Modelled with analytic Gaussian (erf) edges.
Bitrate [Gbps]Line rate (after encoding when an encoder is selected).
PRBS patternMaximum-length LFSR (PRBS-7 … PRBS-31). See §9.2.
EncodingNone / 8b10b / 64b66b / 128b130b / PAM4. See §9.3.

Driver (differential)

Same parameter set as the single-ended driver, but with two outputs in anti-phase. The differential voltage Vdiff = V+ − V− is what the eye-scope reads.

Transient Step Source (single-ended)

  • Amplitude [V] and Polarity (Positive / Negative).
  • Rise / Fall time [s]: 10–90 %, modelled with erf edges.
  • Delay [s]: time origin offset.

Used by the transient simulator (§10) to extract step responses.

Transient Pulse Source (single-ended)

Same parameters as the step source plus Pulse width [s]. Useful for impulse-response approximation and reflectometry-style probes.

Scope (single-ended)

Time-domain probe used by transient simulations. Records the voltage waveform at the connected node.

Scope (differential)

Same as above, but records V+ − V−.

Eye-Scope (single-ended)

Channel-simulation receiver. Folds the received waveform into UI segments and produces an eye diagram with measurements (see §9.7–9.8).

Eye-Scope (differential)

Same as above on the differential signal.

8.2 Network solver (Y / Z / S conversion)

For each frequency in the global sweep the editor builds an indefinite admittance matrix from every component, then condenses it on the external port nodes. Touchstone blocks are first converted from S to Y in their own reference impedance; lumped elements add their analytic contribution; ideal grounds eliminate the corresponding row/column. The condensed Y is then converted back to S using:

S = (Y0 - Y) * inv(Y0 + Y) where Y0 = diag(1 / Z0_k)
From circuit graph to exported Touchstone model.

8.3 Equivalent network export

  • External ports are detected from dedicated port blocks and reordered by user-assigned port number.
  • The exported S-matrix is written to a Touchstone file in the selected format (RI / MA / DB) and frequency unit.
  • Validation runs before export: missing connections, dangling nodes, or empty sweep raise an error dialog.

9. Channel Simulation & Eye Diagram ↑ top

9.1 Pipeline overview

  1. Build the equivalent S-matrix of the network (§8.2).
  2. Generate the raw bit stream from the selected PRBS (§9.2) and apply encoding (§9.3).
  3. Render the bit stream as a smooth NRZ waveform with erf transitions (§9.4).
  4. Compute the voltage transfer function from driver to receiver including driver source impedance and S11 reflection (§9.5).
  5. Convolve the waveform with the impulse response of the transfer function (§9.6).
  6. Inject statistical perturbations (random voltage noise Vrn, optional jitter).
  7. Fold the waveform into UI-length segments and render the eye (§9.7).
  8. Compute the eye-summary metrics (§9.8).

9.2 PRBS generators

PRBS streams are generated with a Fibonacci LFSR in shift-LEFT canonical form, all-ones seed. The taps used are the standard maximum-length polynomials:

PatternTaps (bit positions)Period

PRBS-8[8, 6, 5, 4]255
PRBS-9[9, 5]511
PRBS-10[10, 7]1023
PRBS-11[11, 9]2047
PRBS-12[12, 11, 10, 4]4095
PRBS-13[13, 12, 2, 1]8191
PRBS-15[15, 14]32767
PRBS-20[20, 17]1 048 575
PRBS-23[23, 18]8 388 607
PRBS-31[31, 28]2 147 483 647

A maximum-length sequence has exactly 2N−1 ones, 2N−1−1 zeros, longest run of N ones and N−1 zeros. With the Maximal Length LFSR option enabled, exactly one full period is generated and tiled until the requested num_bits is reached, replicating the ADS Bit Source convention.

9.3 8b/10b encoder (running disparity)

A full Widmer-Franaszek 8b/10b D-character encoder is implemented (IEEE 802.3 Cl.36 tables 36-1 and 36-2). The bit stream is grouped into bytes LSB-first; each byte is split into a 5-bit (EDCBA) and a 3-bit (HGF) sub-block, mapped through the 5b/6b and 3b/4b lookup tables. The current running disparity selects the RD− or RD+ codeword and is updated by the codeword's own disparity. The alternate D.x.A7 form is used to suppress 5-in-a-row across the 6b/4b boundary.

  • Output length: ⌈N/8⌉ · 10 bits (zero-padded last byte if needed).
  • Maximum run length on the wire: 5.
  • DC balance: the long-term average is exactly 0.5 (perfect).
  • Bitrate convention: the value entered in the UI is the line rate, i.e. the rate after encoding.
64b/66b and 128b/130b are currently no-ops (the bit stream is passed through unchanged); a real implementation would add a self-synchronous scrambler with polynomial x58+x39+1 and 2-bit sync header. PAM4 affects only the eye level count.

9.4 NRZ waveform with erf edges

Each bit boundary is rendered with an analytic Gaussian (erf) step centred on the boundary, scaled by the symbol delta. This is the IEEE 802.3 COM convention for finite-rise NRZ stimuli and is significantly more accurate than convolving an ideal rectangular waveform with a moving average. Per-edge cost is O(samples_per_bit).

tr (10-90%) = 2 * sqrt(2) * erfinv(0.8) * sigma ~= 2.5631 * sigma edge(t) = 0.5 * (V_high + V_low) + 0.5 * delta * erf((t - t_edge) / (sqrt(2) * sigma))

9.5 Voltage transfer function (Zsrc + S11)

A standard S-parameter file describes power waves between matched ports. To convert it into a voltage transfer function from a non-ideal driver to a matched probe we have to account for both the source impedance mismatch and the reflection back from the network input port:

H(f) = S21(f) * sqrt(Z0_out * Z0_in) ------------------------------------------------------- (Z0_in + Z_src) + S11(f) * (Z0_in - Z_src)

When Zsrc = Z0,in the denominator collapses to 2·Z0,in and we recover the conventional matched-source voltage transfer. With Zsrc ≠ Z0,in the network input reflection S11 reshapes the transfer because the wave that bounces back interferes with the incident wave at the source.

9.6 Convolution with impulse response

Once H(f) is known on a uniform spectrum, the impulse response h(t) is obtained by inverse real FFT and the receiver waveform is the linear convolution of the transmitted waveform with h(t):

h(t) = F^-1 { H(f) } (impulse response) y(t) = (x * h)(t) = integral x(tau) * h(t - tau) d tau

The convolution is computed with scipy.signal.fftconvolve (overlap-add FFT) for O(N log N) cost. The number of impulse-response samples is sized to capture the first ~10/πfmin seconds of the response so that low-frequency truncation error stays below 1‰.

9.7 Eye folding & jitter injection

  1. The waveform is split into segments of length UI = 1 / Bitrate. A symmetric two-UI window centred on the decision instant is plotted.
  2. Optional Gaussian voltage noise of RMS Vrn is added to every sample.
  3. Optional Gaussian timing jitter is applied as a per-segment time offset (jitter_samples = jitter_rms_ps · 10−12 / dt).
  4. The decision phase is aligned by maximizing the eye opening centred at t = 0.
  5. Rendering quality preset (Fast / Balanced / HighRes) controls the heatmap resolution.

9.8 Measurements

  • One Level / Zero Level: medians of upper / lower amplitude clusters around the decision center.
  • Eye Amplitude: One Level − Zero Level.
  • Eye Height: vertical opening at decision center using percentile envelopes.
  • Eye Width: contiguous open interval around center, reported in ps.
  • Eye Crossing: crossing level measured around ±0.5 UI and normalized to Eye Amplitude (%).
  • RJ(σ): random-jitter standard deviation extracted from the crossing distribution.
  • DJ(pp): deterministic-jitter peak-to-peak from the bimodal histogram fit.
  • TJ(pp)@BER: total jitter at the configured bit-error rate combining DJ and Q·RJ.
EyeAmplitude = OneLevel - ZeroLevel EyeHeight = p5(upper) - p95(lower) at decision center EyeWidth_UI = open_samples / samples_per_UI EyeWidth_ps = EyeWidth_UI * UI * 1e12 EyeCrossing_pct = 100 * (CrossingLevel - ZeroLevel) / EyeAmplitude TJ(pp)@BER = DJ(pp) + 2 * Q(BER) * RJ_sigma
Eye measurement geometry used by the summary panel.

10. Transient Simulation (Step / Pulse Sources) ↑ top

Transient simulations use the same frequency-domain solve as the channel pipeline but excite the network with a deterministic step or pulse stimulus from a transient source block. The procedure is:

  1. Build the equivalent S-matrix at every sweep frequency.
  2. Compute the voltage transfer function from the source port to every scope port (same denominator as §9.5, with the source impedance defaulting to the port reference impedance).
  3. Inverse-FFT to obtain the impulse response h(t).
  4. Convolve the source stimulus (step or pulse) with h(t).
  5. Apply optional Gaussian smoothing matching the requested rise time.

The resulting waveform is shown in the Transient window and can be saved as part of the project.

11. Project Files ↑ top

A saved project stores:

  • Loaded file paths.
  • Plot, TDR, and circuit windows with their settings, sizes, and saved output references.
  • Selected traces, legend labels, hidden plot-local files, and per-window configuration.
  • Generated outputs such as exported circuit models and saved eye snapshots.

When an exported project is reopened, relative source paths inside the export bundle are resolved against the exported project folder automatically.

12. Notes & Troubleshooting ↑ top

  • If a saved project references missing files, loading proceeds with warnings.
  • You are prompted before closing when the project has unsaved changes.
  • If a TDR plot shows excessive ringing, increase the upper frequency or use a stronger taper window.
  • If an eye diagram looks collapsed, verify the driver Vhigh/Vlow levels and the source impedance, and check that Bitrate matches the transmitter under analysis.
  • If 64b/66b or 128b/130b encoding is required for bit-exact replication, note that the current build implements only the 8b/10b code; higher-rate scramblers can be added on request.
