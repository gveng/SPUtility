SPUtility User Guide

SPUtility is a desktop viewer for Touchstone S-parameter files. It lets you load one or more files, inspect tables, compare traces in independent plot windows, and save a full working session as a project.

What You Can Do

  • Open Touchstone files such as .s1p, .s2p, .s4p, and .ts.
  • Open raw data tables and magnitude tables for each loaded file.
  • Create multiple independent graph windows to compare different trace selections.
  • Select which S-parameters are plotted for each file.
  • Edit the legend label used for each loaded file in every plot window.
  • Change plot settings for linear or logarithmic axes and manual or automatic ranges.
  • Save and reopen projects with loaded files, graph selections, labels, and window sizes.
  • Arrange internal windows using tile, cascade, minimize, restore, or resize-all commands.

Main Workflow

  1. Use File > Open File to load one or more Touchstone files.
  2. Use the Tables menu to inspect raw or magnitude data for each loaded file.
  3. Use Charts > Open plot window to create a graph window.
  4. In each graph window, enable the desired S-parameters using the checkboxes.
  5. Edit the Legend label column if you want custom names in the graph legend.
  6. Use Settings > Plot settings inside a graph window to configure axis behavior.
  7. Use File > Save Project to store your current workspace in a JSON project file.

Plot Windows

Each plot window is independent. This means you can:

  • Open several graph windows at the same time.
  • Choose different trace combinations in each window.
  • Use different legend labels per plot window.
  • Resize windows independently or align them using the View menu commands.

TDR Calculation

The TDR window computes a time-domain impedance profile from the selected reflection trace (for example S11) of each loaded Touchstone file.

  1. The selected complex trace is extracted versus frequency.
  2. Frequency samples are sorted and resampled on a uniform frequency grid.
  3. A user-selected frequency-domain window is applied (for example Hann, Hamming, Blackman, Flattop, or None) to control ringing and resolution tradeoff.
  4. An inverse real FFT (SciPy) is used to obtain the impulse response in time domain.
  5. The impulse response is cumulatively integrated to obtain a step reflection profile.
  6. Reflection is converted to impedance using Z(t) = Z0 * (1 + rho) / (1 - rho), with Z0 = 50 Ohm.
  7. The result is plotted as impedance versus time.

In the TDR control bar, you can adjust:

  • Rise time: applies smoothing to emulate finite edge speed.
  • Zero before front: defines a flat 50 Ohm segment before the reflected step appears.
  • Total time: limits the displayed time window.
  • Window: lets you choose the frequency-domain windowing function used before the inverse FFT.

Windowing quick guide

Window When to use it Tradeoff
Hann Good default for most traces. Balanced ringing reduction and edge sharpness.
Hamming When you want slightly stronger sidelobe control than Hann. Smoother response with a small loss of time resolution.
Blackman When ringing suppression is more important than edge detail. Very good sidelobe suppression, softer transitions.
Flattop When amplitude flatness is prioritized. Lowest apparent ripple but widest main lobe (poorer resolution).
None For raw response inspection or maximum edge sharpness. Highest ringing risk and more spectral leakage artifacts.

Project Files

A saved project stores the working session, including:

  • The list of loaded file paths.
  • All open graph windows.
  • All open TDR windows.
  • Selected parameters in each graph window.
  • Legend labels for each file.
  • Plot settings and graph window sizes.
  • TDR file visibility/trace selections, legend labels, window choice, rise time, zero-before-front duration, total time, and TDR window sizes.

View Menu

  • Tile windows: arranges sub-windows in a tiled layout.
  • Cascade windows: stacks sub-windows in cascade order.
  • Minimize all: minimizes every open sub-window.
  • Restore all: restores all minimized sub-windows.
  • Resize all graph with last graph window: applies the current graph layout to the other graph windows.
  • Close all: closes every open sub-window.

Notes

  • If a saved project references files that no longer exist, the project opens with warnings.
  • Graph windows reflect only the traces you selected in that specific window.
  • You are prompted before closing the application if the current project has unsaved changes.
